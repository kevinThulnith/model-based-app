# SEDI Flutter App - Complete Setup Guide

## What You Are Building

A Flutter mobile app that connects to your SEDI Django backend (Diabetes, Heart Disease, CKD prediction APIs), authenticates users with Google Sign-In, and saves all assessment history to Firestore. The design follows the Editorial Wellness system from the screenshots: teal gradients, pill shapes, capsule cards, no divider lines.

## Step 1 - Prerequisites

Install the following before starting:

- Flutter SDK 3.10 or later — https://flutter.dev/docs/get-started/install
- Android Studio with Android SDK (or Xcode for iOS)
- Node.js 18 or later (for FlutterFire CLI)
- A Google account for Firebase

Verify Flutter is installed:

```bash
flutter doctor
```

All items should be checked. If Android toolchain shows issues, open Android Studio, go to SDK Manager, and accept all licenses:

```bash
flutter doctor --android-licenses
```

## Step 2 - Create the Flutter Project

Open a terminal in the directory where you want the project:

```bash
flutter create sedi_app
cd sedi_app
```

Now copy all provided code files into this project, replacing the generated ones. The structure must match exactly what is listed above.

## Step 3 - Create the Firebase Project

1. Go to https://console.firebase.google.com
2. Click "Add project"
3. Name it "sedi-wellness" (or any name you prefer)
4. Disable Google Analytics if you do not need it, then click "Create project"
5. Wait for the project to be created, then click "Continue"

---

## Step 4 - Enable Firebase Authentication

Inside your Firebase project:

1. Click "Authentication" in the left sidebar
2. Click "Get started"
3. Click the "Sign-in method" tab
4. Click "Google" in the provider list
5. Toggle "Enable" to ON
6. Set a support email (use your own Gmail address)
7. Click "Save"

---

## Step 5 - Enable Firestore Database

Inside your Firebase project:

1. Click "Firestore Database" in the left sidebar
2. Click "Create database"
3. Choose "Start in production mode" and click "Next"
4. Select a region closest to your users (e.g., asia-south1 for Sri Lanka) and click "Enable"
5. Once created, click the "Rules" tab
6. Replace the default rules with the content of the provided firestore.rules file:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
      match /assessments/{assessmentId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }
  }
}
```

7. Click "Publish"

## Step 6 - Configure FlutterFire CLI

Install the FlutterFire CLI globally:

```bash
dart pub global activate flutterfire_cli
```

If the command is not found after installation, add the pub cache bin to your PATH:

On Windows:
```
%USERPROFILE%\AppData\Local\Pub\Cache\bin
```

On Mac/Linux:
```
export PATH="$PATH:$HOME/.pub-cache/bin"
```

Now run FlutterFire configure from inside the sedi_app directory:

```bash
flutterfire configure
```

- Select your Firebase project from the list
- Select the platforms you want (choose Android, and iOS if needed)
- This generates lib/firebase_options.dart automatically

Do not edit firebase_options.dart manually. It is auto-generated.

## Step 7 - Configure Google Sign-In for Android

Google Sign-In on Android requires a SHA-1 fingerprint registered in Firebase.

Get your debug SHA-1:

```bash
cd android
./gradlew signingReport
```

On Windows:
```bash
cd android
gradlew signingReport
```

Look for the "debug" variant output. Copy the SHA1 value (looks like: AB:CD:12:34:...).

Register it in Firebase:
1. Go to Firebase Console > Project Settings (gear icon)
2. Scroll down to "Your apps" and click your Android app
3. Click "Add fingerprint"
4. Paste the SHA-1 value and click "Save"
5. Download the updated google-services.json
6. Replace android/app/google-services.json with the downloaded file

---

## Step 8 - Configure Google Sign-In for iOS (if building for iPhone)

1. Go to Firebase Console > Project Settings
2. Click your iOS app
3. Download GoogleService-Info.plist
4. Open the ios/ folder in Xcode:
   ```bash
   open ios/Runner.xcworkspace
   ```
5. Drag GoogleService-Info.plist into the Runner folder in Xcode
6. Make sure "Copy items if needed" is checked and click Finish

Add the URL scheme to ios/Runner/Info.plist. Open the file and add inside the outermost dict:

```xml
<key>CFBundleURLTypes</key>
<array>
  <dict>
    <key>CFBundleTypeRole</key>
    <string>Editor</string>
    <key>CFBundleURLSchemes</key>
    <array>
      <string>YOUR_REVERSED_CLIENT_ID</string>
    </array>
  </dict>
</array>
```

Replace YOUR_REVERSED_CLIENT_ID with the REVERSED_CLIENT_ID value from GoogleService-Info.plist. It looks like: com.googleusercontent.apps.1234567890-abcdef...

## Step 9 - Set Your Backend URL

Open lib/services/api_service.dart and replace the placeholder URL with your actual deployed Django backend:

```dart
static const String _baseUrl = 'https://your-sedi-backend.com';
```

If you are running the backend locally for testing, use:

```dart
static const String _baseUrl = 'http://10.0.2.2:8000';  // Android emulator
// or
static const String _baseUrl = 'http://localhost:8000';  // iOS simulator
```

Note: For Android emulator, 10.0.2.2 maps to your computer's localhost.

## Firestore Data Structure

```
Firestore
  users/
    {userId}/               <- Firebase Auth UID
      assessments/
        {assessmentId}/     <- Auto-generated document ID
          type: "diabetes" | "heart_disease" | "ckd"
          userId: string
          createdAt: Timestamp
          prediction: 0 | 1
          label: "Diabetic" | "Non-Diabetic" | "Presence" | "Absence" | "Affected" | "Not Affected"
          confidence: float  (e.g. 91.1)
          probability: {
            non_diabetic: 0.089,
            diabetic: 0.911
          }
          inputs: {          <- Full form data sent to API
            age: 45,
            bmi: 24.0,
            ...
          }
```

## Common Issues and Fixes

Issue: PlatformException SIGN_IN_FAILED on Android
Fix: The SHA-1 fingerprint is not registered in Firebase, or google-services.json is outdated. Redo Step 7.

Issue: MissingPluginException for firebase_core
Fix: Run `flutter clean` then `flutter pub get` then restart the app.

Issue: CLEARTEXT communication not allowed (Android)
Fix: Your backend URL is HTTP, not HTTPS. Either use HTTPS for your backend, or add this to android/app/src/main/AndroidManifest.xml inside the application tag:
```xml
android:usesCleartextTraffic="true"
```
Only do this for development. Use HTTPS in production.

Issue: GoogleService-Info.plist not found (iOS)
Fix: The file must be added through Xcode (drag and drop), not just placed in the folder.

Issue: flutter_animate causing null assertion errors
Fix: Ensure your Flutter SDK is 3.10 or later with `flutter upgrade`.

Issue: Font not loading
Fix: google_fonts downloads fonts at runtime. Ensure you have internet connectivity, or bundle the fonts locally by downloading Plus Jakarta Sans and Inter from Google Fonts and adding them to pubspec.yaml as assets.

## Customization Points

To change the backend URL: lib/services/api_service.dart, line with `_baseUrl`

To change app name: android/app/src/main/AndroidManifest.xml (android:label) and ios/Runner/Info.plist (CFBundleName)

To change colors: lib/theme/app_theme.dart, the AppColors class

To add more assessment types: create a new screen following the pattern in diabetes_assessment_screen.dart, add a new API method in api_service.dart, and add a model card in home_screen.dart

To add user profile fields: use FirestoreService().saveUserProfile() and getUserProfile() in profile_screen.dart

---

## Security Notes

- Firestore rules ensure each user can only access their own data
- Never put API keys in client-side Flutter code
- The Django backend should be protected with HTTPS in production
- Google Sign-In tokens are managed by Firebase Auth and are not stored manually
- Consider adding rate limiting on the Django backend to prevent abuse
